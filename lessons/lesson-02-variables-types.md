# Урок 2. Переменные, типы данных, выражения

В этом уроке мы разбираем, как Rust представляет данные, какие типы у нас есть «из коробки», как работают переменные и почему почти всё в Rust — это выражение.

## Теория

### Иммутабельность по умолчанию

Объявление переменной выглядит так:

```rust
let x = 5;
```

По умолчанию переменные **неизменяемы**. Попытка изменить значение даст ошибку компиляции:

```rust
let x = 5;
x = 6;          // error: cannot assign twice to immutable variable `x`
```

Чтобы значение можно было менять, добавляют ключевое слово `mut`:

```rust
let mut x = 5;
x = 6;          // ок
```

Иммутабельность по умолчанию — это сознательное проектное решение: компилятор помогает быстро находить места, где значение действительно меняется, и тем самым делает код более предсказуемым. Эта же идея проявляется в системе ссылок (урок 5).

### Shadowing (затенение)

```rust
let x = 5;
let x = x + 1;       // новый биндинг x = 6, прежний забыт
let x = "пять";      // даже тип может смениться
```

Shadowing — это **новое объявление** переменной с тем же именем. Оно отличается от `mut` тремя важными вещами:

1. Создаётся новая переменная: новый тип, новое владение.
2. Старая переменная попадает в `drop` (если у неё был ресурс).
3. Shadowing разрешён без `mut`.

Типичный пример пользы: распарсить строку в число, переиспользовав имя.

```rust
let s = "  42  ";
let s = s.trim();          // &str
let s: i32 = s.parse().unwrap();  // i32
```

### Константы и static

```rust
const MAX_USERS: u32 = 100_000;
static APP_NAME: &str = "myapp";
```

- `const` — компилируется как константа времени компиляции, может быть встроена прямо в место использования. Тип обязателен.
- `static` — реально живёт в секции данных бинаря, имеет адрес и время жизни `'static`.
- Имена пишутся в `SCREAMING_SNAKE_CASE`.
- В `const` и `static` по умолчанию могут вычисляться только const-выражения.

### Скалярные типы

**Целые** числа:

| Размер | Знаковый | Беззнаковый |
|--------|----------|-------------|
| 8 бит  | `i8`     | `u8`        |
| 16 бит | `i16`    | `u16`       |
| 32 бит | `i32`    | `u32`       |
| 64 бит | `i64`    | `u64`       |
| 128    | `i128`   | `u128`      |
| native | `isize`  | `usize`     |

`isize`/`usize` — размером со слово платформы (4 байта на 32-бит, 8 на 64-бит). Используются для индексов и длин массивов. Литералы поддерживают подчёркивания и системы счисления:

```rust
let dec = 1_000_000;        // десятичный
let hex = 0xff;             // шестнадцатеричный
let oct = 0o77;             // восьмеричный
let bin = 0b1010_1010;      // двоичный
let byte = b'A';            // u8 = 65
let typed = 42_i64;         // суффикс типа
```

Без явного типа целочисленный литерал имеет тип `i32`.

**Числа с плавающей точкой**: `f32` и `f64`. По умолчанию литерал — `f64`. Стандартные операции: `+ - * / %`. Полезные методы: `.sqrt()`, `.powi(n)`, `.powf(x)`, `.abs()`, `.floor()`, `.round()`, `.ceil()`.

**bool** — занимает 1 байт, значения `true`/`false`. В Rust нет неявного приведения чисел к `bool` — `if 1 { ... }` не скомпилируется.

**char** — 4 байта, любой Unicode scalar value (диапазон U+0000..U+D7FF и U+E000..U+10FFFF). Это **не байт UTF-8 строки**, а кодовая точка.

```rust
let c1: char = 'a';
let c2: char = 'я';
let c3: char = '🦀';
let c4: char = '\\u{1F600}';  // 😀 через unicode escape
```

### Целочисленные переполнения

В debug-сборке Rust **паникует при переполнении**. В release — делает «обёртку» (wrapping). Чтобы поведение не зависело от профиля, используют явные методы:

```rust
let a: u8 = 250;
let b: u8 = 10;
let c1 = a.wrapping_add(b);              // 4 (255 + 0..)
let c2 = a.checked_add(b);               // None — переполнение
let c3 = a.saturating_add(b);            // 255 — упёрлось в максимум
let (c4, overflow) = a.overflowing_add(b); // (4, true)
```

### Преобразования типов

Неявных приведений нет — пишут `as` или `From`/`Into`/`TryFrom`/`TryInto`:

```rust
let n: i64 = 200;
let m = n as i32;                  // обрезает старшие биты
let f = n as f64;                  // i64 -> f64

let n: i32 = i32::from(123_u8);    // безопасное расширение
let big: i64 = 5_000_000_000;
let small: i32 = i32::try_from(big).unwrap_or(0); // проверка
```

Правило: для безопасных преобразований предпочитайте `From`/`TryFrom`. `as` — мощный, но «тихий», легко получить мусор при downcast.

### Составные типы

**Кортеж** — фиксированный набор значений разных типов:

```rust
let t: (i32, f64, char) = (1, 2.0, 'a');
let (x, y, z) = t;             // деструктуризация
println!("{} {} {}", t.0, t.1, t.2);
let unit: () = ();             // unit-кортеж
```

Тип `()` (unit) представляет «нет значимых данных», как `void` в C, но это полноценный тип.

**Массив** — фиксированный размер на стеке, тип `[T; N]`:

```rust
let a: [i32; 5] = [1, 2, 3, 4, 5];
let zeros = [0u8; 1024];      // 1024 нуля
println!("len={}", a.len());
let first = a[0];             // паника, если выйти за границы
let safe = a.get(10);         // Option<&i32>
```

Размер — часть типа: `[i32; 3]` и `[i32; 4]` — разные типы. Для динамической длины есть `Vec<T>` (урок 6).

### Выражения и блоки

Rust — выражение-ориентированный язык. Это значит, что почти всё имеет тип и значение. Блок `{ ... }` — это выражение: его значением становится последняя строка без `;`.

```rust
let y = {
    let n = 3;
    n * n           // нет ; — это значение блока
};
println!("{}", y);  // 9
```

Если поставить `;`, выражение превратится в инструкцию, и значение станет `()`:

```rust
let y = {
    let n = 3;
    n * n;          // ; превращает в инструкцию
};                  // y: ()
```

`if` — тоже выражение:

```rust
let n = 7;
let parity = if n % 2 == 0 { "even" } else { "odd" };
```

Все ветви `if` должны иметь один тип, иначе компилятор не сможет вывести тип всего выражения.

### Println и Debug

Для пользовательских типов чаще всего удобнее печатать через `Debug`:

```rust
#[derive(Debug)]
struct P(i32, i32);

fn main() {
    let p = P(1, 2);
    println!("{:?}", p);    // P(1, 2)
    println!("{:#?}", p);   // P(\\n    1,\\n    2,\\n)
}
```

Если нужен «человеческий» вид — реализуют `std::fmt::Display` (об этом — в уроке 8).

## Практика

### Задача 1. Статистика массива

```rust
fn stats(arr: [i32; 5]) -> (i32, i32, i32, f64) {
    let min = *arr.iter().min().unwrap();
    let max = *arr.iter().max().unwrap();
    let sum: i32 = arr.iter().sum();
    let avg = sum as f64 / arr.len() as f64;
    (min, max, sum, avg)
}

fn main() {
    let (mn, mx, s, a) = stats([3, 1, 4, 1, 5]);
    println!("min={mn}, max={mx}, sum={s}, avg={a:.2}");
}
```

Дополните: верните также квадрат отклонения от среднего (variance).

### Задача 2. Безопасная работа с переполнением

```rust
fn add_capped(a: u32, b: u32, cap: u32) -> u32 {
    a.saturating_add(b).min(cap)
}

fn main() {
    println!("{}", add_capped(100, 50, 1000));            // 150
    println!("{}", add_capped(u32::MAX, 1, 1_000_000));   // 1_000_000 — упёрлись в cap
    println!("{}", add_capped(900, 200, 1000));           // 1000
}
```

### Задача 3. Конвертер температуры

```rust
fn c_to_f(c: f64) -> f64 { c * 9.0 / 5.0 + 32.0 }
fn f_to_c(f: f64) -> f64 { (f - 32.0) * 5.0 / 9.0 }

fn main() {
    for c in [-40.0, 0.0, 25.0, 100.0_f64] {
        let f = c_to_f(c);
        println!("{c:>6.1}°C = {f:>6.1}°F");
    }
    for f in [-40.0, 32.0, 98.6, 212.0_f64] {
        let c = f_to_c(f);
        println!("{f:>6.1}°F = {c:>6.1}°C");
    }
}
```

Обратите внимание, что `-40°C == -40°F` — это легко проверить запуском.

### Задача 4. Числа Фибоначчи через кортеж

```rust
fn fib(n: u32) -> u64 {
    let mut state: (u64, u64) = (0, 1);
    for _ in 0..n {
        state = (state.1, state.0 + state.1);
    }
    state.0
}

fn main() {
    for i in 0..15 {
        println!("fib({i:>2}) = {}", fib(i));
    }
}
```

Замените тип `u64` на `u32` и подайте `fib(50)` в debug-сборке — получите панику от переполнения. В release — wrap-эффект и неверный результат. Так на практике видно разницу.

### Задача 5. Парсинг с shadowing

```rust
fn parse_age(s: &str) -> Option<u32> {
    let s = s.trim();             // &str
    let s = s.trim_start_matches('+');
    let n: u32 = s.parse().ok()?; // shadowing на u32
    if n > 150 { None } else { Some(n) }
}

fn main() {
    for v in ["  42  ", "+100", "0", "abc", "200", "  18"] {
        println!("{:?} -> {:?}", v, parse_age(v));
    }
}
```

### Задача 6. Проверка переполнения

```rust
fn safe_factorial(n: u32) -> Option<u64> {
    let mut acc: u64 = 1;
    for i in 1..=n as u64 {
        acc = acc.checked_mul(i)?;
    }
    Some(acc)
}

fn main() {
    for n in [0, 5, 10, 20, 21, 25] {
        println!("{n}! = {:?}", safe_factorial(n));
    }
}
```

`21!` уже не помещается в `u64` — `checked_mul` вернёт `None`, и `?` пробросит его наружу.

---

### Ответы

1. `mut` позволяет изменять значение по тому же биндингу — тип не меняется, владение — то же. Shadowing создаёт **новую переменную** с тем же именем: меняется тип, владение, старая попадает в `drop`. Shadowing разрешён без `mut`.

2. 4 байта. `char` хранит любую Unicode scalar value (до `U+10FFFF`); этот диапазон не влезает в 1 или 2 байта. Это не байт UTF-8 — это абстрактная кодовая точка.

3. `12`. Каждое `let x = ...` создаёт новый биндинг: 5 → 6 → 12.

4. Скомпилируется. Shadowing разрешён независимо от `mut` и может менять тип переменной. `mut` от первого `let` становится неактуален, потому что новый `let` затеняет переменную совсем.

5. `y` имеет тип `()`: после `n * 2` стоит `;`, поэтому блок возвращает unit. Чтобы получить число, нужно убрать `;` у последней строки.

6. Не скомпилируется: `+` требует, чтобы оба операнда были одного типа. `1.0` — это `f64`, `2` — `i32` (по умолчанию). Нужно явно: `1.0 + 2.0` или `1.0 + (2 as f64)`.

7. `wrapping_add` — арифметика по модулю, переполнение «оборачивается». `checked_add` — возвращает `Option`: `None` при переполнении. `saturating_add` — упирается в максимум/минимум типа. `overflowing_add` — возвращает `(результат, флаг_переполнения)`.

8. В Rust условие `if` строго типизировано: должен быть `bool`. Неявных преобразований чисел в `bool` нет — это убирает классические ошибки вроде `if (x = 5)` и заставляет писать намерение явно: `if x != 0`.

9. Напечатает `20 -1`. Индексация `a[1]` даёт `20`. `a.get(10)` за пределами массива — `None`; `copied()` превращает `Option<&i32>` в `Option<i32>`, `unwrap_or(-1)` возвращает `-1`.

10. `const` — это значение времени компиляции; компилятор может инлайнить его в точки использования, у него нет одного фиксированного адреса. `static` живёт в одном месте в памяти всё время работы программы, имеет адрес и тип `'static`. `static mut` существует, но требует `unsafe` для доступа.
# Урок 2. Переменные, типы данных, выражения

## Теория

### Иммутабельность по умолчанию

Объявление переменной: `let x = 5;`. По умолчанию переменные неизменяемы. Чтобы изменять — нужно `let mut x = 5;`. Это сознательное проектное решение: компилятор помогает находить места, где значение меняется.

### Shadowing (затенение)

```rust
let x = 5;
let x = x + 1;       // новая переменная x, прежняя забыта
let x = "пять";      // даже тип может смениться
```

Shadowing отличается от `mut`: создаётся новый биндинг с новым типом и владением, прежний попадает в drop.

### Константы

```rust
const MAX_USERS: u32 = 100_000;
```

Тип обязателен. Вычисляются на этапе компиляции, доступны как константные выражения. `static` — другая сущность: имеет адрес в памяти и время жизни `'static`.

### Скалярные типы

Целые: `i8 i16 i32 i64 i128 isize` и `u8..u128 usize`. Литералы: `1_000`, `0xff`, `0b1010`, `b'A'`. Размер по умолчанию для целочисленных литералов — `i32`.

Числа с плавающей точкой: `f32`, `f64` (по умолчанию).

`bool` — 1 байт. `char` — 4 байта, любой Unicode scalar value (U+0000..U+D7FF и U+E000..U+10FFFF).

### Составные типы

Кортежи: `let t: (i32, f64, char) = (1, 2.0, 'a');`, доступ `t.0`.

Массивы фиксированного размера: `let a: [i32; 5] = [1,2,3,4,5];`. Размер — часть типа.

### Выражения и блоки

Rust — выражение-ориентированный язык. Блок `{ ... }` — это выражение: возвращает значение последней строки без `;`.

```rust
let y = {
    let n = 3;
    n * n           // обратите внимание — нет ;
};
```

`if` тоже выражение: `let s = if cond { "yes" } else { "no" };`.

## Практика

Реализуйте функцию, считающую статистику массива из 5 чисел:

```rust
fn stats(arr: [i32; 5]) -> (i32, i32, i32) {

    (min, max, sum)
}

fn main() {
    let (mn, mx, s) = stats([3, 1, 4, 1, 5]);
    println!("min={mn}, max={mx}, sum={s}");
}
```

Дополните: добавьте подсчёт среднего как `f64`, верните кортеж из 4 элементов и распечатайте через именованный форматтер `{name}`.

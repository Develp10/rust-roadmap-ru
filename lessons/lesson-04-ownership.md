# Урок 4. Ownership (владение)

## Теория

### Три правила

1. У каждого значения есть один владелец.
2. В каждый момент времени владелец один.
3. Когда владелец выходит из scope — значение дропается (вызывается `Drop`).

Это исключает double-free, use-after-free и data races на этапе компиляции, без сборщика мусора.

### Move vs Copy

Для типов, реализующих `Copy` (целые, `bool`, `char`, `f64`, кортежи из `Copy`), присваивание копирует биты:

```rust
let a = 5;
let b = a; // a всё ещё валиден
```

Для не-`Copy` (`String`, `Vec`, `Box` и пр.) — происходит **move**:

```rust
let s1 = String::from("hi");
let s2 = s1;       // s1 перемещён
// println!("{}", s1); // ошибка компиляции
```

Передача в функцию — тоже move (или копирование для Copy):

```rust
fn take(s: String) { /* ... */ } // забирает владение
let x = String::from("a");
take(x);
// x недоступен дальше
```

### Clone

`.clone()` делает явное глубокое копирование. Это всегда видимо в коде — не происходит "молча".

### Drop

В конце scope значения дропаются в **обратном порядке** объявления. Можно реализовать `impl Drop` для пользовательских освобождений (файлы, lock'и).

## Практика

Напишите функцию с move и без:

```rust
fn take_and_give_back(s: String) -> String {
    println!("внутри: {}", s);
    s
}

fn make_greeting(name: &str) -> String {
    format!("Привет, {}", name)
}

fn main() {
    let s = String::from("мир");
    let s = take_and_give_back(s); // получаем владение обратно
    let g = make_greeting("Алиса"); // не отдаём владение строкой
    println!("{} | {}", s, g);
}
```

## Тест

**1.** Почему этот код не компилируется?
```rust
fn main() {
    let s1 = String::from("a");
    let s2 = s1;
    println!("{}", s1);
}
```

**2.** Реализует ли `&T` трейт `Copy`? А `&mut T`?

**3.** Что произойдёт?
```rust
fn main() {
    let v = vec![1, 2, 3];
    let f = move || println!("{:?}", v);
    f();
    // println!("{:?}", v); // что здесь?
}
```

**4.** В каком порядке дропнутся `a`, `b`, `c`?
```rust
fn main() {
    let a = String::from("a");
    let b = String::from("b");
    let c = String::from("c");
}
```

**5.** Скомпилируется ли?
```rust
fn main() {
    let s = String::from("x");
    let t = s;
    let s = s; // !
}
```

---

### Ответы

1. `s1` перемещён в `s2`; использование после move запрещено. `String` не реализует `Copy`, потому что владеет аллокацией на куче.

2. `&T` реализует `Copy` — это просто указатель, безопасно копировать. `&mut T` **не** реализует `Copy`, иначе нарушалось бы правило "одна изменяемая ссылка".

3. `move` переместит `v` в замыкание; раскомментированная строка не скомпилируется. Без `move` замыкание захватило бы по ссылке, и `v` был бы доступен.

4. В обратном порядке: `c`, `b`, `a`. Это важно для RAII-ресурсов (mutex guard и т.п.).

5. Не скомпилируется: после `let t = s;` переменная `s` перемещена и недоступна. Чтобы переиспользовать имя, нужно `let s = t;` или `let s = String::from(...)`.

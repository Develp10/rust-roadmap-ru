# Урок 15. Макросы

Макросы в Rust — это способ писать код, который пишет код. Они выполняются на этапе компиляции и позволяют избежать повторений, создавать DSL и расширять синтаксис языка.

## Теория

### Что такое макрос?

Макрос — это правило преобразования одного куска кода в другой. В отличие от функций, макросы:

- Принимают переменное число аргументов разных типов.
- Раскрываются на этапе компиляции, а не вызываются в рантайме.
- Могут генерировать произвольный код.

В Rust есть два больших класса макросов:

1. **Декларативные макросы** (`macro_rules!`) — описываются паттернами и подстановками.
2. **Процедурные макросы** — это код на Rust, который принимает токены и возвращает токены. К ним относятся `#[derive(...)]`, attribute-макросы и function-like макросы.

### macro_rules! — декларативные макросы

Самый простой макрос:

```rust
macro_rules! say_hello {
    () => {
        println!("Привет, Rust!");
    };
}

fn main() {
    say_hello!();
}
```

Синтаксис `macro_rules!` напоминает `match`: слева — паттерн, справа — раскрытие.

### Захват аргументов: фрагменты

В паттернах используются *фрагменты* со спецификаторами:

| Спецификатор | Что захватывает              |
|--------------|------------------------------|
| `expr`       | выражение                    |
| `ident`      | идентификатор                |
| `ty`         | тип                          |
| `pat`        | паттерн                      |
| `stmt`       | выражение-инструкция         |
| `block`      | блок `{ ... }`               |
| `item`       | элемент (fn, struct и т.д.)  |
| `literal`    | литерал                      |
| `tt`         | одно дерево токенов          |

Пример с захватом выражения:

```rust
macro_rules! square {
    ($x:expr) => {
        $x * $x
    };
}

fn main() {
    let n = square!(5);          // 25
    let m = square!(2 + 3);      // ВНИМАНИЕ: 2 + 3 * 2 + 3 = 11, а не 25!
    println!("{n} {m}");
}
```

Это классическая ошибка: при подстановке надо оборачивать аргумент в скобки:

```rust
macro_rules! square {
    ($x:expr) => {
        ($x) * ($x)
    };
}
```

### Повторители `$( ... )*`

Макросы умеют принимать произвольное число аргументов:

```rust
macro_rules! my_vec {
    ( $( $x:expr ),* $(,)? ) => {{
        let mut v = Vec::new();
        $( v.push($x); )*
        v
    }};
}

fn main() {
    let v: Vec<i32> = my_vec![1, 2, 3, 4];
    println!("{v:?}");
}
```

Разбор паттерна:

- `$( $x:expr ),*` — ноль и более выражений через запятую.
- `$(,)?` — необязательная завершающая запятая.
- `$( v.push($x); )*` — повторить блок столько раз, сколько было аргументов.

### Гигиена

Макросы в Rust *гигиеничны*: переменные, объявленные внутри макроса, не конфликтуют с переменными снаружи. Это важное отличие от макросов в C.

```rust
macro_rules! make_x {
    () => { let x = 10; };
}

fn main() {
    let x = 1;
    make_x!();
    println!("{x}"); // 1, а не 10
}
```

### Экспорт макросов

Чтобы использовать макрос в другом крейте, его помечают атрибутом `#[macro_export]`:

```rust
#[macro_export]
macro_rules! debug_print {
    ($($arg:tt)*) => {
        if cfg!(debug_assertions) {
            eprintln!($($arg)*);
        }
    };
}
```

### Процедурные макросы

Процедурные макросы — это отдельные крейты с типом `proc-macro = true`. Они работают с потоком токенов `TokenStream` и обычно используют крейты `syn` (парсер) и `quote` (генератор).

Виды процедурных макросов:

1. **Derive-макросы** — `#[derive(MyTrait)]` автоматически реализует трейт.
2. **Attribute-макросы** — `#[my_attr]` оборачивают/изменяют элемент.
3. **Function-like** — `my_macro!(...)` принимает произвольный синтаксис.

Скелет derive-макроса:

```rust
// Cargo.toml в крейте macros:
// [lib]
// proc-macro = true

use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput};

#[proc_macro_derive(Hello)]
pub fn derive_hello(input: TokenStream) -> TokenStream {
    let ast = parse_macro_input!(input as DeriveInput);
    let name = &ast.ident;

    let expanded = quote! {
        impl #name {
            pub fn hello() {
                println!("Hello from {}", stringify!(#name));
            }
        }
    };
    expanded.into()
}
```

### Как отлаживать макросы

- `cargo expand` (из crate `cargo-expand`) — показывает раскрытие макросов.
- `trace_macros!(true);` — на nightly печатает шаги раскрытия.
- Стандартный приём — выводить простые сообщения через `compile_error!("сообщение")` для проверки путей.

## Практика

Напишем макрос `hashmap!`, аналог `vec!`, но для `HashMap`:

```rust
use std::collections::HashMap;

macro_rules! hashmap {
    ( $( $key:expr => $value:expr ),* $(,)? ) => {{
        let mut map = HashMap::new();
        $( map.insert($key, $value); )*
        map
    }};
}

fn main() {
    let scores = hashmap! {
        "Аня" => 95,
        "Боря" => 80,
        "Вася" => 72,
    };

    for (name, score) in &scores {
        println!("{name}: {score}");
    }

    // макрос можно использовать и без trailing comma
    let empty: HashMap<&str, i32> = hashmap!{};
    println!("Пустая карта: {empty:?}");
}
```

И ещё пример — макрос-логгер с уровнями:

```rust
macro_rules! log {
    (info, $($arg:tt)*) => { println!("[INFO]  {}", format_args!($($arg)*)) };
    (warn, $($arg:tt)*) => { println!("[WARN]  {}", format_args!($($arg)*)) };
    (error, $($arg:tt)*) => { eprintln!("[ERROR] {}", format_args!($($arg)*)) };
}

fn main() {
    log!(info, "сервис стартовал на порту {}", 8080);
    log!(warn, "медленный ответ: {} мс", 1500);
    log!(error, "не удалось открыть файл {:?}", "config.toml");
}
```

## Тест

1. Чем макрос отличается от обычной функции?
2. Что захватывает спецификатор `expr`, а что — `tt`?
3. Почему в раскрытии макроса `square!` важно оборачивать `$x` в скобки?
4. Что делает `#[macro_export]`?
5. Какой макрос подойдёт, если нужно автоматически реализовать трейт для пользовательской структуры — декларативный или процедурный derive-макрос?

---

### Ответы

1. Функция вызывается в рантайме и принимает строго типизированные аргументы. Макрос работает на этапе компиляции, может принимать переменное число аргументов разных категорий и генерировать произвольный код.
2. `expr` захватывает корректное Rust-выражение целиком (например, `2 + 3` или `foo(x).bar()`). `tt` — это «дерево токенов»: один токен или сбалансированно скобочная группа; самый гибкий, но и самый «низкоуровневый» спецификатор.
3. Без скобок подстановка попадает прямо в выражение и приоритеты операторов ломают логику: `square!(2 + 3)` раскроется в `2 + 3 * 2 + 3 = 11`. Скобки сохраняют группировку и дают ожидаемый результат.
4. `#[macro_export]` экспортирует декларативный макрос на уровень крейта, делая его доступным для использования через `use crate_name::macro_name`.
5. Процедурный derive-макрос: он умеет анализировать структуру и генерировать `impl`-блок. Декларативный `macro_rules!` так не умеет, потому что не имеет доступа к произвольному анализу типов.
